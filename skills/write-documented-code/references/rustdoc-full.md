# Rustdoc Full Reference — Conventions, Cargo Doc Config, and Documentation Architecture

## Cargo Doc Configuration

Rust documentation is generated by `rustdoc`, invoked through `cargo doc`. There
is no standalone config file like Doxyfile or typedoc.json. Configuration happens
through `Cargo.toml` fields, `.cargo/config.toml` flags, and crate-level
attributes.

### Cargo.toml Documentation Fields

```toml
[package]
name = "nirapod-core"
version = "0.1.0"
edition = "2021"
description = "Core cryptographic primitives and device protocol for Nirapod hardware wallets."
documentation = "https://docs.nirapod.dev/nirapod-core"
repository = "https://github.com/nirapod/nirapod-core"
license = "MIT"
readme = "README.md"
keywords = ["hardware-wallet", "crypto", "ble", "nrf52840"]
categories = ["cryptography", "embedded", "hardware-support"]

# docs.rs-specific configuration (used when publishing to crates.io)
[package.metadata.docs.rs]
all-features = true
rustdoc-args = ["--cfg", "docsrs"]
```

### .cargo/config.toml — Persistent Rustdoc Flags

```toml
# .cargo/config.toml
[build]
rustdocflags = [
    # Fail on any rustdoc warning (zero-warnings policy)
    "-D", "warnings",
    # Custom CSS for Nirapod branding (optional)
    "--extend-css", "docs/custom.css",
]
```

### Crate-Level Lint Attributes

Every `lib.rs` or `main.rs` in the Nirapod project must enable these lints:

```rust
// src/lib.rs — top of file, after the license header

#![warn(missing_docs)]
#![warn(rustdoc::missing_crate_level_docs)]
#![warn(rustdoc::broken_intra_doc_links)]
#![warn(rustdoc::private_intra_doc_links)]
#![warn(clippy::missing_docs_in_private_items)]
```

`missing_docs` is the equivalent of Doxygen's `WARN_IF_UNDOCUMENTED`. Combined
with `-D warnings` in rustdocflags, the build breaks if any public item lacks
documentation.

### Generating and Viewing Docs

```bash
# Generate docs for the current crate (no dependencies)
cargo doc --no-deps

# Generate and open in browser
cargo doc --no-deps --open

# Generate with private items visible (for internal review)
cargo doc --no-deps --document-private-items

# Run only the doctests
cargo test --doc
```

## Comment Syntax Reference

Rust has two documentation comment styles and their attribute equivalents.

### Outer Doc Comments: `///`

Used to document the item that follows. Placed immediately before the item.

```rust
/// Encrypts plaintext using AES-256-GCM with the specified key handle.
pub fn encrypt_gcm(key: &KeyHandle, plaintext: &[u8]) -> Result<Vec<u8>, CryptoError> {
    // ...
}
```

### Inner Doc Comments: `//!`

Used to document the enclosing item (the crate or module). Placed at the top of
a file, inside the item.

```rust
//! BLE transport layer for Nirapod device communication.
//!
//! This module handles service discovery, GATT characteristic subscriptions,
//! and packet framing over Bluetooth Low Energy.
```

### Doc Attributes: `#[doc]` and `#![doc]`

`///` is syntactic sugar for `#[doc = "..."]`. You rarely need the attribute form,
but it is useful for:

- **Conditional docs:** `#[cfg_attr(docsrs, doc(cfg(feature = "ble")))]`
- **Hiding items:** `#[doc(hidden)]` keeps an item public but hides it from docs
- **Aliases:** `#[doc(alias = "aes_encrypt")]` makes the item findable by alias

## Standard Documentation Sections

Rustdoc does not have formal tags like TSDoc's `@param` or `@returns`. Instead,
use standard Markdown headings within the doc comment. The Rust API Guidelines
define these sections:

### `# Panics`

Document the conditions under which a function panics. Any function that can
panic on user-supplied input MUST document this.

```rust
/// # Panics
///
/// Panics if `index` is greater than or equal to `self.len()`.
```

### `# Errors`

Document the conditions under which a function returns `Err`. Any function
returning `Result` MUST have this section.

```rust
/// # Errors
///
/// Returns [`CryptoError::KeyExpired`] if the key handle has been invalidated
/// by a rotation event.
///
/// Returns [`CryptoError::InputTooLong`] if `plaintext.len()` exceeds 65535
/// bytes (CC310 DMA limit).
```

### `# Safety`

For `unsafe` functions, this section is **mandatory**. It documents the
preconditions the caller must uphold to avoid undefined behavior.

```rust
/// # Safety
///
/// The caller must ensure that:
/// - `ptr` is a valid, aligned pointer to a `NirapodPacket`.
/// - The pointed-to memory is not concurrently mutated.
/// - The pointer remains valid for the duration of this call.
```

### `# Examples`

Every public item should have at least one example. Examples are compiled and
tested by `cargo test`. Use `# ` (hash-space) at the start of a line to hide
setup code from the rendered docs while keeping it in the doctest.

```rust
/// # Examples
///
/// ```
/// use nirapod_core::crypto::encrypt_gcm;
/// use nirapod_core::key::KeyHandle;
///
/// # let key = KeyHandle::test_key();
/// let plaintext = b"hello world";
/// let ciphertext = encrypt_gcm(&key, plaintext)?;
/// assert!(!ciphertext.is_empty());
/// # Ok::<(), nirapod_core::CryptoError>(())
/// ```
```

## Crate-Level Documentation Template

Place this at the top of `src/lib.rs`, after the SPDX license header.

```rust
// SPDX-License-Identifier: APACHE-2.0
// SPDX-FileCopyrightText: 2026 Nirapod Contributors

//! # Nirapod Core
//!
//! Cryptographic primitives and device protocol for Nirapod hardware wallets.
//!
//! This crate provides the host-side implementation of the Nirapod protocol,
//! including AES-256-GCM encryption, ECDSA-P256 signing, BIP-32 key derivation,
//! and transport-layer communication with Nirapod hardware devices over BLE
//! and USB-HID.
//!
//! ## Architecture
//!
//! ```text
//! [Application]
//!       |
//! [nirapod_core]  <-- this crate
//!       |
//! +-----+-------------------+
//! |                         |
//! [BLE Transport]     [USB-HID Transport]
//! ```
//!
//! ## Quick Start
//!
//! ```no_run
//! use nirapod_core::{NirapodClient, BleTransport};
//!
//! # async fn example() -> Result<(), Box<dyn std::error::Error>> {
//! let transport = BleTransport::connect().await?;
//! let client = NirapodClient::new(transport);
//! let address = client.get_address("m/44'/60'/0'/0/0").await?;
//! println!("Device address: {}", address);
//! # Ok(())
//! # }
//! ```
//!
//! ## Feature Flags
//!
//! | Feature   | Description                               | Default |
//! |-----------|-------------------------------------------|---------|
//! | `ble`     | BLE transport via btleplug                | yes     |
//! | `usb`     | USB-HID transport via hidapi              | yes     |
//! | `mock`    | In-memory mock transport for testing      | no      |
//!
//! ## Modules
//!
//! - [`crypto`] — AES-GCM, ECDSA, key derivation
//! - [`transport`] — Device communication layer
//! - [`protocol`] — Wire format parsing and serialization
//! - [`error`] — Error types and result aliases

#![warn(missing_docs)]
#![warn(rustdoc::missing_crate_level_docs)]
#![warn(rustdoc::broken_intra_doc_links)]
```

## Module-Level Documentation Template

Place `//!` docs at the top of each module file.

```rust
// SPDX-License-Identifier: APACHE-2.0
// SPDX-FileCopyrightText: 2026 Nirapod Contributors

//! AES-256-GCM encryption and decryption.
//!
//! This module wraps the platform's AES-GCM implementation and provides a
//! unified interface regardless of whether the backend is ring, RustCrypto,
//! or a hardware-accelerated FFI binding.
//!
//! All key material is handled through opaque [`KeyHandle`] references.
//! Raw key bytes are never exposed through this API.
//!
//! # Examples
//!
//! ```
//! use nirapod_core::crypto::aes::{encrypt_gcm, decrypt_gcm};
//! use nirapod_core::key::KeyHandle;
//!
//! # let key = KeyHandle::test_key();
//! let plaintext = b"secret message";
//! let sealed = encrypt_gcm(&key, plaintext)?;
//! let opened = decrypt_gcm(&key, &sealed)?;
//! assert_eq!(&opened, plaintext);
//! # Ok::<(), nirapod_core::CryptoError>(())
//! ```
```

## Struct Documentation Template

```rust
/// 32-byte packed secure payload for BLE and USB-HID transmission.
///
/// Defines the wire format shared between all Nirapod firmware targets and the
/// TypeScript host application (`@nirapod/protocol`). All multi-byte fields are
/// **little-endian** (native ARM byte order).
///
/// This packet is authenticated with AES-128-GCM; the `auth_tag` field carries
/// the 8-byte truncated GCM tag. The receiver must verify `auth_tag` before
/// processing any field. Discard the packet on tag mismatch.
///
/// # Examples
///
/// ```
/// use nirapod_core::protocol::NirapodPacket;
///
/// let raw = [0u8; 32];
/// let packet = NirapodPacket::from_bytes(&raw)?;
/// assert_eq!(packet.version, 1);
/// # Ok::<(), nirapod_core::ProtocolError>(())
/// ```
#[repr(C, packed)]
pub struct NirapodPacket {
    /// Protocol version. Currently `0x01`.
    pub version: u8,
    /// Message type. See [`MessageType`] for valid values.
    pub msg_type: u8,
    /// Sequence number. Wraps at 65535.
    pub seq: u16,
    /// Milliseconds since device boot. Wraps at ~49 days.
    pub timestamp_ms: u32,
    /// Encrypted payload. Format depends on `msg_type`.
    pub payload: [u8; 16],
    /// Truncated AES-128-GCM authentication tag (bytes 0 through 7).
    pub auth_tag: [u8; 8],
}
```

## Enum Documentation Template

```rust
/// Identifies which cryptographic backend is active on the current platform.
///
/// Selected at compile time via feature flags, but queryable at runtime through
/// [`CryptoBackend::current()`]. The value is included in the device attestation
/// report so the host can verify hardware acceleration is in use on production
/// hardware.
///
/// # Examples
///
/// ```
/// use nirapod_core::crypto::CryptoBackend;
///
/// let backend = CryptoBackend::current();
/// println!("Using: {:?}", backend);
/// ```
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum CryptoBackend {
    /// ring library (AEAD, ECDSA) with assembly-optimized primitives.
    Ring,
    /// RustCrypto crates (aes-gcm, p256) with pure-Rust implementations.
    RustCrypto,
    /// FFI binding to the CC310 hardware engine on nRF52840.
    #[cfg(feature = "nrf52840")]
    Cc310Hw,
    /// FFI binding to the CC312 via PSA Crypto on nRF5340.
    #[cfg(feature = "nrf5340")]
    Cc312Psa,
}
```

## Trait Documentation Template

```rust
/// Transport-layer abstraction for communicating with a Nirapod hardware device.
///
/// Implementations handle the physical/link layer details (BLE GATT, USB-HID
/// reports, or mocks for testing). The trait exposes a simple send/receive model
/// operating on raw [`NirapodPacket`] instances.
///
/// Implementations must handle reconnection internally. If the transport drops,
/// [`DeviceTransport::send`] must return [`TransportError::Disconnected`] rather
/// than silently failing.
///
/// # Examples
///
/// ```
/// use nirapod_core::transport::{DeviceTransport, MockTransport};
/// use nirapod_core::protocol::NirapodPacket;
///
/// # async fn example() -> Result<(), Box<dyn std::error::Error>> {
/// let mut transport = MockTransport::new();
/// let packet = NirapodPacket::ping();
/// transport.send(&packet).await?;
/// # Ok(())
/// # }
/// ```
#[async_trait]
pub trait DeviceTransport: Send + Sync {
    /// Sends a packet to the connected device.
    ///
    /// # Errors
    ///
    /// Returns [`TransportError::Disconnected`] if the device is not connected.
    /// Returns [`TransportError::Timeout`] if the send does not complete within
    /// the configured timeout (default: 5000 ms).
    async fn send(&mut self, packet: &NirapodPacket) -> Result<(), TransportError>;

    /// Receives the next packet from the device.
    ///
    /// # Errors
    ///
    /// Returns [`TransportError::Disconnected`] if the device disconnects while
    /// waiting for a packet.
    async fn receive(&mut self) -> Result<NirapodPacket, TransportError>;
}
```

## Function Documentation Template

```rust
/// Encrypts plaintext using AES-256-GCM with the specified key handle.
///
/// Dispatches to the active crypto backend (ring, RustCrypto, or hardware FFI).
/// The function generates a random 12-byte nonce internally via the platform
/// CSPRNG and prepends it to the ciphertext output. The caller does not need to
/// manage nonces separately.
///
/// The output format is: `[12-byte nonce][ciphertext][16-byte GCM tag]`.
/// Use [`decrypt_gcm`] to reverse this operation.
///
/// # Errors
///
/// Returns [`CryptoError::KeyExpired`] if the key handle has been invalidated
/// by a rotation event.
///
/// Returns [`CryptoError::InputTooLong`] if `plaintext.len()` exceeds 65535
/// bytes (CC310 DMA limit on nRF52840 targets).
///
/// Returns [`CryptoError::RngFailure`] if the CSPRNG cannot produce a nonce.
///
/// # Panics
///
/// Panics if the crypto backend was not initialized. Call [`init()`] at
/// application startup before using any crypto functions.
///
/// # Examples
///
/// ```
/// use nirapod_core::crypto::aes::{encrypt_gcm, decrypt_gcm};
/// use nirapod_core::key::KeyHandle;
///
/// # let key = KeyHandle::test_key();
/// let plaintext = b"transaction payload";
/// let sealed = encrypt_gcm(&key, plaintext)?;
///
/// // sealed contains: [nonce (12)][ciphertext (19)][tag (16)] = 47 bytes
/// assert_eq!(sealed.len(), 12 + plaintext.len() + 16);
///
/// let opened = decrypt_gcm(&key, &sealed)?;
/// assert_eq!(&opened, plaintext);
/// # Ok::<(), nirapod_core::CryptoError>(())
/// ```
///
/// # See Also
///
/// - [`decrypt_gcm`] for the corresponding decryption function.
/// - [`KeyHandle`] for how to obtain a valid key handle.
pub fn encrypt_gcm(key: &KeyHandle, plaintext: &[u8]) -> Result<Vec<u8>, CryptoError> {
    // ...
}
```

## Doctest Reference

Rustdoc compiles and runs code examples as tests. This is one of Rust's most
powerful documentation features.

### Hidden Lines

Lines starting with `# ` (hash-space) are compiled and tested but hidden from
the rendered documentation. Use them for setup code, imports, and error handling
that would clutter the example.

```rust
/// ```
/// # use nirapod_core::crypto::aes::encrypt_gcm;
/// # use nirapod_core::key::KeyHandle;
/// # let key = KeyHandle::test_key();
/// let ciphertext = encrypt_gcm(&key, b"hello")?;
/// # Ok::<(), nirapod_core::CryptoError>(())
/// ```
```

### Doctest Annotations

Use these annotations on the code fence to control test behavior:

- **` ```no_run `** — Compiles but does not execute. Use for examples that
  require hardware or network access.
- **` ```should_panic `** — Expected to panic. Use for examples demonstrating
  error conditions.
- **` ```compile_fail `** — Expected to fail compilation. Use for examples
  showing what NOT to do.
- **` ```ignore `** — Skipped entirely by `cargo test`. Use as a last resort
  for examples that cannot be compiled in isolation.
- **` ```text `** — Not compiled at all. Use for ASCII diagrams, protocol
  formats, and non-Rust content.

## Intra-Doc Linking

Rustdoc automatically resolves links to other items in the crate or standard
library when using bracket syntax.

```rust
/// Uses the [`KeyHandle`] from the [`key`] module.
/// See [`CryptoError::KeyExpired`] for the error variant.
/// See [`std::vec::Vec`] for the output container.
```

For methods and associated items, use the full path:

```rust
/// Call [`NirapodClient::connect`] before using this method.
```

For disambiguation between functions, types, and macros with the same name:

```rust
/// See [`encrypt_gcm()`] (the function, not the type).
```

## Conditional Documentation

### `#[cfg(doc)]`

Rustdoc sets the `doc` cfg flag when building documentation. Use this to include
items in docs that are normally gated behind features or targets.

```rust
#[cfg(any(feature = "ble", doc))]
pub mod ble;
```

### `#[doc(cfg(...))]`

Adds a visual badge to the docs indicating the required feature or platform.
Requires nightly or the `docsrs` cfg.

```rust
#[cfg(feature = "ble")]
#[cfg_attr(docsrs, doc(cfg(feature = "ble")))]
pub struct BleTransport { /* ... */ }
```

When rendered, the docs show: "Available on crate feature `ble` only."

## Doc Aliases

Make items searchable by alternative names. Useful when the Rust API uses
different naming conventions than other language implementations.

```rust
#[doc(alias = "aes_encrypt")]
#[doc(alias = "AesDriver_encryptGcm")]
pub fn encrypt_gcm(/* ... */) { /* ... */ }
```

Users searching the docs for "aes_encrypt" (the C function name) will find the
Rust equivalent.
